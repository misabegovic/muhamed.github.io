# User / customer personas

Add one markdown file per user or customer archetype this pi-brain instance cares about.

Name the file after the persona, e.g. `ops-engineer.md` or `non-technical-pm.md`.

Each file should include:

- **Name** — a memorable label.
- **Context** — their role, constraints, and goals.
- **Pain points** — what frustrates them today.
- **Success looks like** — how they know the product helped.

The PM Agent references these personas when framing PRDs and pitches.
